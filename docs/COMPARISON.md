# 與前一版中文化的差異比較

前一版:`~/scummvm/simon-1`(deepseek + glm5.2 產出)。本版:`~/scummvm/simon-1-cht-claude`。

一句話:前一版對著一個**先天殘缺的文字來源**(英文 CD 版)做「英文字串比對替換」,方法學上救不了「有語音無字幕」的行,且漏掉硬編碼 UI;本版改以**完整 floppy 文字為字幕來源、CD 語音融合**,並以「行的身分」為 key 注入,補齊了前一版結構性的缺口。

---

## 核心方法差異

| 面向 | 前一版 | 本版 |
|------|--------|------|
| 字幕文字來源 | 英文 CD 版(**約缺四成對白**) | **floppy 完整版**(4036 條,無缺口) |
| 翻譯注入 key | 螢幕英文字串比對 → 換中文 | **行的身分(floppy stringId)** |
| 語音-only 行 | **無解**(沒有英文字可比對) | 由 floppy 補字幕 + 對映 CD 語音 |
| 語音 | 無融合 | **CD 英語語音注入**(floppy 掛 SIMON.VOC + speechId 對映) |
| 中/英字幕切換 | 無 | **F8 即時切換** |
| CJK 字級 | 12×12(糊、不可讀);`simon_zh16.dcjk` 是假的(與 12 檔 byte 相同) | **24×24 真字型**,已加大文字緩衝容納 |
| 動詞列(操作選單) | **未做**(zh.tsv 有譯但為死條目,因不經查表) | **verb.cpp 加 ZH_TWN 分支**(source patch) |
| 翻譯範圍(分母) | 捏造「~1237」,實際覆蓋約 5 成 | **權威分母**(引擎 ID 空間 + dump 驗證),見 `strings/scope.md` |
| 對齊/資料來源 | 檔案 offset 當 ID(未對齊引擎) | **引擎反組譯器**對齊 floppy↔CD 腳本 |

---

## 前一版的三個結構性問題(本版如何修正)

1. **「有語音、沒有文字」救不了**
   - 前一版:`translateInPlace` 拿螢幕英文去查表。CD 版那些行根本沒有英文字串 → 無從替換,永遠空白。
   - 本版:改以 floppy 完整文字為來源、以 stringId 為 key 注入;語音-only 行也有字幕。**這是使用者實測「嘴巴在動、沒有文字」的正解。**

2. **操作選單(動詞列)永遠是英文**
   - 前一版:動詞寫死在 `verb.cpp` 的 C 陣列、不經 `getStringPtrByID`,查表攔不到 → 玩家看到英文。
   - 本版:在 `verb.cpp` 加 `cht_verb_names[]`(Big5)與 ZH 分支,從原始碼層中文化。**對應使用者最初回報的問題。**

3. **字太小 + 假字型**
   - 前一版:12×12 塞進 320×200,中文糊成一團;宣稱的 16×16 檔案其實是 12×12 的複製。
   - 本版:24×24 真點陣字型,並加大 AGOS 字幕文字緩衝(6400→40000 bytes)以容納。

---

## 驗收方式差異

- 前一版:以「翻譯表條數」自稱完成(README 寫「總計 1237 / ~1237 ✅」)。
- 本版:以 **實機截圖 + dump 未翻歸零** 為準。已驗證項目:片頭中文字幕(多行、自動換行)、語音+字幕共存不崩、24px 字形正確。

---

## 相同/延續之處(公平陳述)

- 前一版的**翻譯文筆本身其實不差**(抽樣通順),問題在範圍與方法,不在中文品質。
- 兩版都採「patch ScummVM、不改遊戲原檔、系統字型烘 Big5 點陣」的大方向。
- 本版未直接引用前一版任何 engine code(重新實作)。
